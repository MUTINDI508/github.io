Learn how to set up Jekyll, a simple, blog-aware static site generator, on your system to quickly build static websites. This guide covers the essential steps for installation and getting your first Jekyll site running.

Jekyll is a static site generator that takes plain text files written in Markdown, runs them through Liquid templates, and produces a complete, static website ready to be served by Apache, Nginx, or GitHub Pages. It's an excellent tool for blogs, personal websites, and documentation sites.

Prerequisites

  • Ruby (version 2.7.0 or higher is recommended)
  • RubyGems (usually comes with Ruby)
  • GCC and Make (for compiling native extensions, common on Linux/macOS, requires DevKit on Windows)

Installation Steps

  1. Install Ruby:
    • macOS: Ruby is often pre-installed. You can use a version manager like `rbenv` or `RVM` for better control.
    • Linux: Use your distribution's package manager (e.g., `sudo apt install ruby-full` on Debian/Ubuntu).
    • Windows: Use RubyInstaller for Windows. Make sure to check the box for "Add Ruby executables to your PATH" during installation. If you're on Windows, also install the **DevKit** from RubyInstaller to compile native gems.
  2. Install Jekyll and Bundler: Once Ruby is installed, open your terminal or command prompt and run: 
    gem install jekyll bundler
    This command installs Jekyll and Bundler, a gem manager that helps manage your project's Ruby dependencies.
  3. Create a new Jekyll site: Navigate to the directory where you want to create your site and run: 
    jekyll new my-awesome-blog
    Replace `my-awesome-blog` with your desired project name. This command creates a new directory with a basic Jekyll site structure.
  4. Navigate into your site: 
    cd my-awesome-blog
  5. Serve the site: To see your site in action, run the Jekyll server: 
    bundle exec jekyll serve
    This command builds your site and starts a local server. You can usually view your site at `http://localhost:4000`. Jekyll will automatically recompile your site when you make changes to your files.

You now have Jekyll installed and a basic site running! You can start customizing the content, themes, and layouts to build your perfect website.
