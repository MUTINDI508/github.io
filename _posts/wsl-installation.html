Install and configure Windows Subsystem for Linux (WSL) to run a Linux environment directly within Windows without the overhead of a traditional VM. This guide simplifies the setup process.

Windows Subsystem for Linux (WSL) allows developers to run a GNU/Linux environment directly on Windows, without the overhead of a traditional virtual machine or dual-boot setup. It's a game-changer for Windows users who need access to Linux command-line tools, utilities, and applications.

Why Use WSL?

  • Developer Tools: Access Linux-specific tools like Git, Node.js, Python, Docker, and various compilers directly from Windows.
  • Performance: WSL 2, in particular, offers significant performance improvements over WSL 1 and traditional VMs for file system operations.
  • Seamless Integration: Easily access your Windows files from within Linux, and vice-versa.
  • Lightweight: Faster startup times and less resource-intensive than full virtual machines.

Installation Steps (WSL 2 Recommended)

Ensure your Windows 10 version is 2004 or higher (Build 19041+) or Windows 11 for the best experience with WSL 2.

  1. Enable WSL (via PowerShell): Open PowerShell as an administrator and run the following command. This will enable the necessary optional components. 
    dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
    Restart your computer when prompted.
  2. Set WSL 2 as your default version: After restarting, open PowerShell as administrator again and run: 
    wsl --set-default-version 2
    If you're upgrading from WSL 1, you might need to convert existing distributions: `wsl --set-version <Distro Name> 2`.
  3. Install a Linux Distribution: Open the Microsoft Store and search for your preferred Linux distribution (e.g., "Ubuntu", "Debian", "Kali Linux"). Click "Get" or "Install".
  4. Launch and Setup: Once installed, launch the distribution from the Start Menu. The first time you run it, it will take a few minutes to set up. You'll be prompted to create a username and password for your new Linux environment.
  5. Update and Upgrade (Optional but Recommended): Inside your Linux distribution, it's good practice to update your packages: 
    sudo apt update && sudo apt upgrade

You can now run Linux commands and install Linux applications directly from your Windows machine. This integration drastically simplifies the development workflow for many cross-platform projects.
